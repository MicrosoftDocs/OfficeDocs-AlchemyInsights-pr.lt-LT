# <a name="permissions-and-public-folders"></a>Teisės ir viešieji aplankai

Viešuosiuose aplankuose galite keisti teises naudodami "Outlook", "Exchange" administravimo centrą (EAC) arba "PowerShell":
  
- Jei norite gauti "Outlook" instrukcijas, [spustelėkite čia](https://support.office.com/article/Set-or-change-permissions-for-a-public-folder-b2e0440c-7873-48ec-9ff2-b1a20b723005.aspx).
    
- Dėl EAC, ieškokite [šiame straipsnyje](https://technet.microsoft.com/library/jj651147%28v=exchg.150%29.aspx.aspx#Anchor_1) instrukcijas. 
    
- "PowerShell", ieškokite [šiame straipsnyje](https://technet.microsoft.com/library/bb124743%28v=exchg.160%29.aspx.aspx) instrukcijas, kaip naudoti Add-PublicFolderClientPermission komandų. Jei jums reikia instrukcijų, kaip prisijungti prie "Exchange PowerShell", spustelėkite [čia](https://technet.microsoft.com/library/jj984289%28v=exchg.160%29.aspx.aspx).
    
Jei **išoriniai vartotojai negali siųsti el. laiškų į viešąjį aplanką, kuriame įgalintas paštas,** gali būti, kad viešajame aplanke trūksta teisių, reikalingų išoriniam el. pašto pristatymui. Tai galite išspręsti naudodami "Outlook" instrukcijas [čia](https://technet.microsoft.com/library/aa997560%28v=exchg.150%29.aspx.aspx#Anchor_1)arba "PowerShell" instrukcijas [čia](https://support.microsoft.com/help/2984402/-5.7.1-smtp-550-5.7.1-resolver.rst.authrequired-nondelivery-report-when-external-users-try-to-send-mail-to-mail-enabled-public-folders-in-office-365.aspx).
